# Audit Vercel - ops-portfolio

## 🎯 Checklist Déploiement

### 1. CORRIGER LES VERSIONS DES DÉPENDANCES
- [ ] Installer `npm-check-updates` : `npm install -g npm-check-updates`
- [ ] Vérifier les versions : `ncu`
- [ ] Mettre à jour : `ncu -u`
- [ ] Installer : `npm install`
- [ ] Tester build local : `npm run build`

### 2. CONFIGURER LES VARIABLES D'ENVIRONNEMENT VERCEL
Dans Vercel Project Settings → Environment Variables, ajouter :

```
RESEND_API_KEY=<your_resend_api_key>
CONTACT_TO=<your_email_for_contact_form>
CONTACT_FROM=noreply@yourdomain.com
NEXT_PUBLIC_SITE_URL=https://yourdomain.vercel.app
```

### 3. OPTIONNEL : ACTIVER ESLint AU BUILD
Modifier `next.config.mjs` :

```js
eslint: { ignoreDuringBuilds: false },
```

Puis corriger les erreurs ESLint :
```bash
npm run lint -- --fix
```

### 4. VÉRIFIER LA CONFIGURATION TAILWIND
- [ ] Vérifier `tailwind.config.ts` compatible avec version actuelle
- [ ] Tester build CSS : `npm run build`

### 5. VÉRIFIER LE BUILD LOCAL
```bash
npm run build
npm start
```

Vérifier qu'il n'y a pas d'erreurs :
- TypeScript ✓
- ESLint ✓
- CSS compilation ✓
- API routes ✓

## 🚀 DÉPLOYER SUR VERCEL

### Option 1 : Via CLI
```bash
npm i -g vercel
vercel
```

### Option 2 : Via GitHub (recommandé)
1. Push sur GitHub
2. Connecter le repo à Vercel
3. Vercel détectera automatiquement Next.js
4. Configurer les env vars
5. Déployer
